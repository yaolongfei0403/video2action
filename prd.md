# 📋 产品需求文档（PRD）

## 4D 动作知识库系统（Action Knowledge Base System）

---

## 一、项目概述

### 1.1 项目背景

当前项目 `app.py` 是一个基于 **Gradio** 的单视频、人工交互式算法 Demo（SAM-Body4D），核心流程为：

> **上传视频 → 人工点击标注（正/负点） → 多目标添加 → SAM-3 Mask 传播 → 遮挡补全（Diffusion-VAS） → 4D 人体重建 → 渲染输出**

该 Demo 已验证了从视频到 4D 结构化人体数据的端到端算法可行性，但存在以下瓶颈：

| 维度 | 现状 | 问题 |
|------|------|------|
| **输入** | 单视频手动上传 | 无法批量接入，无视频源管理 |
| **切片** | 无 | 长视频需人工挑选有效片段 |
| **交互** | 全人工点击标注 | 效率低，不可规模化 |
| **输出** | 4D 渲染视频 + mesh 文件 | 缺乏语义化标签，无法被下游系统检索引用 |
| **存储** | 本地文件系统 | 无向量检索能力，无法支撑语义搜索 |

### 1.2 项目目标

将单视频算法 Demo 升级为 **"4D 动作知识库系统"**，实现：

```
┌─────────────────────────────────────────────────────────────────┐
│                    4D 动作知识库系统 全景架构                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌────────────┐  │
│  │  上游     │──▶│  核心处理 │──▶│  下游     │──▶│  应用层     │  │
│  │ 视频接入  │   │ 4D重建   │   │ 语义化入库│   │ 文本检索   │  │
│  └──────────┘   └──────────┘   └──────────┘   └────────────┘  │
│                                                                 │
│  GUID批量拉取    打点/Mask/4D    VLM描述+人工标签   纯文本语义检索│
│  粗切(自动化)    渲染视频导出    Qwen3-Embedding    大模型视频引用│
│  精切(人工校验)  对象存储        Weaviate向量库     动作参考匹配  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**核心目标**：构建一个"大模型可引用的动作知识库"，通过纯文本语义检索，为视频大模型提供高质量的 **4D 渲染视频** 作为直接的动作视觉参考。

## 三、功能模块详细设计

### 模块一：视频接入与下载（上游）

#### 3.1.1 GUID 批量视频拉取

| 属性 | 说明 |
|------|------|
| **功能描述** | 用户输入一个或多个 GUID（逗号分隔），系统自动从视频源下载对应视频 |
| **输入** | `guid_list: List[str]`，如 `"abc123,def456,ghi789"` |
| **输出** | 下载到对象存储的原始视频文件，状态更新到任务数据库 |
| **优先级** | P0 |

**数据模型：**

```python
class VideoSource:
    id: str                    # 系统唯一ID (UUID)
    guid: str                  # 原始GUID
    status: Enum               # PENDING / DOWNLOADING / DOWNLOADED / FAILED
    file_path: str             # 对象存储路径 (原始视频)
    duration: float            # 时长(秒)
    resolution: Tuple[int,int] # 分辨率
    fps: float                 # 帧率
    created_at: datetime
```

#### 3.1.2 自动化粗切（Auto Rough-Cut）

| 属性 | 说明 |
|------|------|
| **功能描述** | 对下载的长视频进行自动化切片，筛选出符合规则的片段 |
| **粗切规则** | ① 片段时长 ≥ 5秒 ② 画面存在明显动作（光流/帧差检测） ③ 画面中检测到人体 |
| **输出** | 候选片段列表（Candidate Clips），状态标记为 `ROUGH_CUT` |
| **优先级** | P0 |

#### 3.1.3 人工精切（Manual Fine-Cut）

| 属性 | 说明 |
|------|------|
| **功能描述** | 对粗切候选片段进行人工校验和精细裁剪 |
| **操作** | ① 片段预览与审核（通过/拒绝） ② 时间轴精确裁剪（入点/出点微调） ③ 画面区域裁剪（ROI指定） |
| **输出** | 精切后的最终片段，状态标记为 `FINE_CUT`，进入核心处理流水线 |
| **优先级** | P0 |

### 模块二：核心 4D 处理流水线（基于现有 app.py）

#### 3.2.1 视频打点标注

| 属性 | 说明 |
|------|------|
| **功能描述** | 对精切后的片段进行人体/目标标注，复用现有 `on_click` + `add_target` 逻辑 |
| **增强点** | ① 支持半自动标注（SAM 预标注 + 人工修正） ② 标注进度持久化到数据库 |
| **优先级** | P0 |

#### 3.2.2 Mask 传播与遮挡补全

| 属性 | 说明 |
|------|------|
| **功能描述** | 复用现有 `on_mask_generation()` 的 SAM-3 传播 + Diffusion-VAS 补全逻辑 |
| **增强点** | ① 批处理支持（多片段并行） ② 中间结果持久化 ③ 断点续传 |
| **优先级** | P0 |

#### 3.2.3 4D 人体重建与渲染视频导出

| 属性 | 说明 |
|------|------|
| **功能描述** | 复用现有 `on_4d_generation()` 完成 4D 重建，**不进行复杂的动作参数抽取**，直接将 4D mesh 渲染为视频并上传存储 |
| **输出** | 渲染后的 4D 动作视频文件（MP4），存储至 MinIO/OSS |
| **优先级** | P0 |

**处理流程：**

```
Mask 补全结果 + 原始视频
       │
       ▼
┌──────────────────────────────┐
│ 4D 人体重建 (SAM-3D-Body)    │
│ ─ 生成逐帧 3D Mesh           │
└──────────────────────────────┘
       │
       ▼
┌──────────────────────────────┐
│ Mesh 渲染 (Rendering)        │
│ ─ 将 3D Mesh 渲染回 2D 画面  │
│ ─ 支持原视频叠加或纯色背景    │
│ ─ 合成最终渲染视频 (MP4)      │
└──────────────────────────────┘
       │
       ▼
┌──────────────────────────────┐
│ 视频上传存储                  │
│ ─ 上传至 MinIO/OSS           │
│ ─ 记录 rendered_video_url    │
└──────────────────────────────┘
```

**数据模型更新：**

```python
class Clip:
    # ... 原有字段 ...
    rendered_video_url: str    # 4D 渲染后的视频存储 URL (核心引用资产)
```

### 模块三：语义化与知识库构建（下游）

#### 3.3.1 基于关键词的 VLM 描述生成

| 属性 | 说明 |
|------|------|
| **功能描述** | 用户为视频指定核心关键词，系统调用 **Qwen-VL (7B级别)** 结合视频画面和关键词，生成结构化的动作描述 |
| **输入** | 渲染后的视频片段 + 用户输入的关键词（如：“太极”、“转身”、“缓慢”） |
| **输出** | 结构化的文本描述（JSON 格式） |
| **模型选型** | **Qwen-VL** (如 Qwen2.5-VL-7B-Instruct) |
| **优先级** | P0 |

**VLM 描述生成 Pipeline：**

```
渲染视频 + 用户关键词 (如: "篮球", "投篮", "跳跃")
       │
       ▼
┌──────────────────────────────────────────────────────┐
│  Qwen-VL 视频理解与描述生成                           │
│                                                      │
│  Prompt 模板:                                        │
│  "请仔细观看这段视频。用户提供的核心关键词是：         │
│   【{keywords}】。                                   │
│   请结合视频画面和关键词，从以下维度详细描述人物动作： │
│   1. 动作概述（一句话总结）                           │
│   2. 动作细节（肢体运动轨迹、发力方式）               │
│   3. 动作节奏（速度变化、停顿）                       │
│   4. 适用场景（该动作适合用于什么类型的视频生成）      │
│   请以 JSON 格式输出。"                               │
│                                                      │
└──────────────────────────────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────────────────────┐
│  结构化输出解析                                       │
│  {                                                   │
│    "summary": "人物进行标准的篮球跳投动作...",         │
│    "detail": "双腿微屈起跳，右手托球，左手扶球...",    │
│    "rhythm": "起跳迅速，最高点有短暂滞空停顿...",      │
│    "use_case": "体育竞技、篮球教学、运动品牌广告"      │
│  }                                                   │
└──────────────────────────────────────────────────────┘
```

#### 3.3.2 人工标签补充与校验

| 属性 | 说明 |
|------|------|
| **功能描述** | 人工对 VLM 生成的描述进行审核、修正，并补充专业标签 |
| **操作** | ① 审核/修改 VLM 描述 ② 添加业务标签 ③ 标记数据质量等级 |
| **优先级** | P1 |

                    ┌────▼─────┐

                    ┌────▼──────┐

### 4.2 任务调度策略

| 阶段 | 调度方式 | 并行度 | 资源需求 |
|------|---------|--------|---------|
| 视频下载 | 异步队列 | 高 (IO密集) | 网络带宽 |
| 自动化粗切 | GPU Worker | 中 | GPU (轻量推理) |
| 人工精切/打点 | 人工操作 | - | Web工作台 |
| Mask传播/4D重建 | GPU Worker | 低 (单GPU) | GPU (SAM-3 大模型) |
| Qwen-VL 描述生成 | GPU Worker / API | 中 | GPU (Qwen-VL 7B) |
| Qwen3-Embedding | GPU Worker / API | 高 | GPU (轻量文本模型) |
| Weaviate入库 | 异步队列 | 高 | CPU |

## 八、与现有 app.py 的映射关系

| app.py 现有模块 | 系统化改造方向 |
|----------------|---------------|
| `init_runtime()` | → 改为系统级模型服务，支持多GPU、模型池管理 |

| `RUNTIME` 全局字典 | → 替换为 Redis + PostgreSQL 状态管理 |
| `OUTPUT_DIR` 本地文件 | → 替换为 MinIO/OSS 对象存储 (存储原始视频与渲染视频) |
