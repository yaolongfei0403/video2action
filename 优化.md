## 1. Target Studio · 首帧标注 布局重构 (2026-06-16)

**问题**
- 操作按钮分散: `添加 Target` 在头部, `完成 → 4D` 在左列底部
- 左列堆 4 个 section (点类型/当前 Target/所有 Target/操作流程),emoji 提示文字噪点多
- 画布 `min-height:380px / max-height:420px` 偏小,标注精度受限
- 候选帧 `repeat(4, 1fr)` + `max-height:200px` 拥挤
- 右列 Clip 卡片样式通用,选中态视觉弱
- Marker 居中靠 inline `−8px` 偏移

**改动**
- `frontend/js/modules/target.js`: 重写 `render()`, 整合顶部条 + 三栏 workspace
- `frontend/css/modules.css`: 追加 `.tg-*` 样式块 (顶部条 / 三栏 / 画布 / 候选帧带 / Clip 列表 / Marker)

**新布局**
```
┌──────────────────────────────────────────────────────────────┐
│ ⌖ Target Studio · 首帧标注 | 当前Clip徽章 | [刷新][添加][完成→4D] │
├──────────┬─────────────────────────────┬────────────────────┤
│ 点类型    │ 标注画布 · 分辨率徽章         │ 待标注 Clips (300) │
│ [+正][-负]│ ┌───────────────────────┐   │ ▣ clip 待/已标注   │
│ 当前 Obj │ │     [点击位置]          │   │   时长 · 标注数     │
│ 所有 Obj │ └───────────────────────┘   │                    │
│ 操作提示  │ 候选帧带 (#编号 + 时间 + 徽章) │                    │
└──────────┴─────────────────────────────┴────────────────────┘
```

**约束 (未破坏)**
- API 调用 `listPendingTargets / getClip / extractFrames / annotateClip / listTargets` 不变
- Marker 坐标计算 `offsetX + a.x*scaleX` 保留,居中改用 `transform: translate(-50%,-50%)`
- 业务逻辑 (点击/撤销/清空/添加 Target/完成→4D) 不变
